# 回合制游戏练习项目

## 1. 新建工程

添加资源，上传git出错了。

### 问题根源

**主要问题：Git pack size 超过 GitHub 2GB 限制**

- UE项目有大量大型资源文件（.uasset等），虽然配置了Git LFS，但**历史提交中的大文件仍然作为普通Git对象存储**。
- 当推送时，Git会打包所有对象，包括历史中的大文件，导致超过2GB限制

### 具体原因分析

1. **LFS配置时机问题**
   - 先添加了文件，后配置LFS
   - 已经提交的大文件不会自动转换为LFS
2. **Git工作机制**
   - LFS对象上传成功（3GB）
   - 但Git仓库本身（包含历史）仍然太大
   - 松散对象（loose objects）在推送时会被打包

### 解决方案

1. **`git lfs migrate import`** - 将历史中的大文件迁移到LFS
2. **`git gc --aggressive`** - 强制垃圾回收，清理旧对象
3. **结果：2.77GB → 271KB**

### 教训

**正确的UE项目Git设置流程应该是：**

1. 先创建 `.gitignore` 和 `.gitattributes`
2. 配置好所有LFS规则
3. 然后才添加和提交文件

这样可以避免大文件进入Git历史，从一开始就使用LFS管理。
